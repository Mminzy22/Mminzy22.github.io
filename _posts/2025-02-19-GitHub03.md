## 1. 깃허브 액션과 YAML 파일 개요

GitHub Actions는 YAML 파일을 이용해 워크플로우를 설정합니다. YAML은 들여쓰기로 계층을 표현하는 형식으로, 깃허브 액션에서는 실행 환경과 단계를 정의하는 데 사용됩니다.

### 1.1. YAML 파일의 기본 구조

```yaml
name: 워크플로우 이름  # GitHub Actions의 워크플로우(Workflow) 이름을 지정합니다.
on: 이벤트 트리거 설정  # 이 워크플로우를 실행할 조건을 정의합니다.
jobs:
  작업명:
    runs-on: 실행 환경  # 해당 작업을 실행할 환경을 지정합니다.
    steps:
      - name: 단계 이름  # 각 단계의 설명을 작성합니다.
        uses: 액션 또는 커맨드 실행  # 특정 액션을 실행하거나 직접 명령어를 실행할 수 있습니다.
```

## 2. YAML 파일 구성 요소

### 2.1. `name`: 워크플로우 이름 지정

```yaml
name: Python CI/CD Pipeline  # 실행되는 CI/CD 파이프라인의 이름을 설정합니다.
```

### 2.2. `on`: 이벤트 트리거 정의

```yaml
on:
  push:
    branches:
      - main  # main 브랜치에 push가 발생할 때 실행합니다.
  pull_request:
    branches:
      - main  # main 브랜치에 대한 Pull Request가 생성될 때 실행합니다.
```

### 2.3. `jobs`: 실행할 작업 정의

```yaml
jobs:
  build:
    runs-on: ubuntu-latest  # 최신 Ubuntu 환경에서 실행합니다.
    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v3  # 저장소의 코드를 가져옵니다.
```

### 2.4. `runs-on`: 실행 환경 지정

```yaml
runs-on: ubuntu-latest  # 최신 Ubuntu 버전에서 작업이 실행됩니다.
```

### 2.5. `steps`: 개별 실행 단계 설정

```yaml
steps:
  - name: Python 환경 설정
    uses: actions/setup-python@v4
    with:
      python-version: '3.9'  # Python 3.9 버전을 사용하도록 설정합니다.
```

## 3. YAML 파일 작성 예제

### 3.1. 기본 Python 테스트 CI

```yaml
name: Python CI  # 워크플로우 이름을 설정합니다.

on:
  push:
    branches: [main]  # main 브랜치에 push할 때 실행됩니다.

jobs:
  test:
    runs-on: ubuntu-latest  # 최신 Ubuntu 환경에서 실행합니다.
    steps:
      - uses: actions/checkout@v3  # 저장소 코드를 가져옵니다.
      - uses: actions/setup-python@v4
        with:
          python-version: '3.9'  # Python 3.9 버전을 사용합니다.
      - run: pip install -r requirements.txt  # 필요한 패키지를 설치합니다.
      - run: pytest  # 테스트를 실행합니다.
```

### 3.2. Django + PostgreSQL 연동

```yaml
name: Django CI  # Django 프로젝트의 CI 설정

on: [push, pull_request]  # Push 및 PR 발생 시 실행됩니다.

jobs:
  test:
    runs-on: ubuntu-latest  # 최신 Ubuntu 환경에서 실행합니다.
    services:
      postgres:
        image: postgres:latest  # PostgreSQL 최신 버전을 컨테이너로 실행합니다.
        env:
          POSTGRES_USER: user  # PostgreSQL 사용자 계정을 설정합니다.
          POSTGRES_PASSWORD: password  # 비밀번호를 설정합니다.
          POSTGRES_DB: testdb  # 사용할 데이터베이스를 설정합니다.
        ports:
          - 5432:5432  # 포트 매핑을 설정합니다.
    steps:
      - uses: actions/checkout@v3  # 저장소의 코드를 가져옵니다.
      - uses: actions/setup-python@v4
        with:
          python-version: '3.9'  # Python 3.9 버전 설정
      - run: pip install -r requirements.txt  # 필요한 패키지 설치
      - run: python manage.py migrate  # Django 마이그레이션 수행
      - run: python manage.py test  # Django 테스트 실행
```

### 3.3. 캐싱을 활용한 빌드 속도 개선

```yaml
steps:
  - name: Cache dependencies  # 캐시를 활용하여 의존성 설치 시간을 단축합니다.
    uses: actions/cache@v3
    with:
      path: ~/.cache/pip  # pip 캐시를 저장할 경로 지정
      key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}  # requirements.txt 기반 해시 키 생성
      restore-keys: |
        ${{ runner.os }}-pip-  # 캐시 복원 키 설정
```

## 4. 고급 YAML 작성법

### 4.1. 매트릭스를 활용한 다중 환경 테스트

```yaml
strategy:
  matrix:
    python-version: [3.7, 3.8, 3.9]  # 여러 버전의 Python에서 테스트를 수행하도록 설정합니다.
```

### 4.2. 시크릿 사용 (환경 변수)

```yaml
env:
  SECRET_KEY: ${{ secrets.SECRET_KEY }}  # GitHub Secrets에 저장된 값을 환경 변수로 설정합니다.
```

## 5. 결론

YAML 파일을 활용하면 GitHub Actions에서 다양한 CI/CD 자동화 작업을 수행할 수 있습니다. 이를 통해 빌드, 테스트, 배포 과정을 체계적으로 관리할 수 있으며, 효율적인 개발 프로세스를 구축할 수 있습니다.
