## 1. 깃허브 액션과 YAML 파일 개요

**GitHub Actions**는 CI/CD 자동화를 설정할 수 있는 강력한 도구로, CI(Continuous Integration, 지속적 통합)를 구현하는 데 자주 사용됩니다. CI는 개발자의 코드 변경 사항을 자동으로 테스트하고 빌드하여 품질을 보장하는 프로세스입니다.

YAML 파일을 활용하면 GitHub Actions에서 다양한 CI/CD 자동화 작업을 설정할 수 있으며, 이를 통해 빌드, 테스트, 배포 과정을 효율적으로 관리할 수 있습니다.


## 2. CI 구축의 핵심 요소

### 2.1. 자동화 테스트
코드의 품질을 유지하고 예상치 못한 오류를 방지하기 위해 다양한 테스트 프레임워크(Pytest, Jest, JUnit 등)를 실행할 수 있습니다.

### 2.2. 코드 스타일 및 린트 검사
코드 스타일 검사(예: flake8, ESLint, Pylint)를 설정하여 일관된 코드 스타일을 유지할 수 있습니다.

### 2.3. 빌드 및 아티팩트 생성
애플리케이션을 실행 가능한 상태로 패키징하는 과정이며, 빌드된 결과물을 저장소 또는 외부 서비스(예: AWS S3, GitHub Releases)에 업로드할 수 있습니다.


## 3. YAML 파일의 기본 구조

```yaml
name: 워크플로우 이름  # GitHub Actions의 워크플로우(Workflow) 이름을 지정합니다.
on: 이벤트 트리거 설정  # 이 워크플로우를 실행할 조건을 정의합니다.
jobs:
  작업명:
    runs-on: 실행 환경  # 해당 작업을 실행할 환경을 지정합니다.
    steps:
      - name: 단계 이름  # 각 단계의 설명을 작성합니다.
        uses: 액션 또는 커맨드 실행  # 특정 액션을 실행하거나 직접 명령어를 실행할 수 있습니다.
```

### 3.1. 기본 CI 파이프라인 설정

```yaml
name: Python CI Pipeline  # 워크플로우 이름 설정

on:
  push:
    branches: [main]  # main 브랜치에 push할 때 실행
  pull_request:
    branches: [main]  # main 브랜치에 대한 PR 생성 시 실행

jobs:
  test:
    runs-on: ubuntu-latest  # 실행 환경을 최신 Ubuntu로 설정
    steps:
      - name: 코드 체크아웃  # 저장소의 코드를 가져옴
        uses: actions/checkout@v3
      
      - name: Python 환경 설정  # Python 3.9 버전 사용 설정
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      
      - name: 의존성 설치  # 필요한 Python 패키지 설치
        run: |
          pip install -r requirements.txt
      
      - name: 테스트 실행  # Pytest를 이용한 테스트 수행
        run: pytest
```

### 3.2. Django + PostgreSQL 연동

```yaml
name: Django CI  # Django 프로젝트의 CI 설정

on: [push, pull_request]  # Push 및 PR 발생 시 실행

jobs:
  test:
    runs-on: ubuntu-latest  # 최신 Ubuntu 환경에서 실행
    services:
      postgres:
        image: postgres:latest  # PostgreSQL 최신 버전을 컨테이너로 실행
        env:
          POSTGRES_USER: user  # PostgreSQL 사용자 계정 설정
          POSTGRES_PASSWORD: password  # 비밀번호 설정
          POSTGRES_DB: testdb  # 사용할 데이터베이스 지정
        ports:
          - 5432:5432  # PostgreSQL의 기본 포트를 매핑
    steps:
      - uses: actions/checkout@v3  # 저장소 코드 체크아웃
      - uses: actions/setup-python@v4
        with:
          python-version: '3.9'  # Python 3.9 버전 설정
      - run: pip install -r requirements.txt  # 필요한 패키지 설치
      - run: python manage.py migrate  # Django 마이그레이션 수행
      - run: python manage.py test  # Django 테스트 실행
```

### 3.3. 캐싱을 활용한 성능 개선

```yaml
steps:
  - name: Python 패키지 캐싱  # 패키지 설치 시간 단축을 위한 캐싱 설정
    uses: actions/cache@v3
    with:
      path: ~/.cache/pip  # pip 패키지 캐시 저장 경로 지정
      key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}  # 캐시 키 생성
      restore-keys: |
        ${{ runner.os }}-pip-  # 캐시 복원 키 설정
```


## 4. CI 구축의 고급 기능

### 4.1. 병렬 실행
테스트 및 빌드를 병렬로 실행하면 시간을 단축할 수 있습니다.

```yaml
strategy:
  matrix:
    python-version: [3.7, 3.8, 3.9]  # 다양한 Python 버전에서 테스트 실행
```

### 4.2. 시크릿 사용 (환경 변수 설정)

```yaml
env:
  SECRET_KEY: ${{ secrets.SECRET_KEY }}  # GitHub Secrets에서 환경 변수 로드
```

### 4.3. 빌드 및 아티팩트 업로드

```yaml
  build:
    runs-on: ubuntu-latest
    steps:
      - name: 코드 체크아웃  # 저장소 코드 다운로드
        uses: actions/checkout@v3
      
      - name: 빌드 실행  # 프로젝트 빌드 수행
        run: |
          python setup.py sdist bdist_wheel
      
      - name: 빌드 결과 업로드  # 빌드된 파일을 GitHub Actions의 아티팩트로 저장
        uses: actions/upload-artifact@v3
        with:
          name: build-artifacts
          path: dist/
```


## 5. 결론

GitHub Actions를 활용하면 YAML 설정을 통해 손쉽게 CI/CD를 구축할 수 있습니다. 

- **자동화 테스트**를 설정하여 코드 품질을 유지하고,
- **린트 검사**로 코드 스타일을 점검하며,
- **빌드 및 아티팩트 업로드**를 통해 배포를 원활하게 수행할 수 있습니다.
