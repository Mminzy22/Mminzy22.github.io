## 1. 프로젝트 개요

**프로젝트명** : **AInfo** (맞춤형 공공서비스 추천 AI 챗봇)

**프로젝트 기간** : 2025년 2월 27일 ~ 2025년 3월 30일

**팀원** : 양주영, 박민지, 정지웅, 유종열, 채희경

**프로젝트 배경**

정부 및 공공기관에서 제공하는 복지, 창업, 고용, 교육 관련 정책 정보는 방대하지만, 정보가 여러 기관에 분산되어 있어 개인이 원하는 정보를 찾기 어렵습니다. 이에 따라 필요한 정책을 모르고 신청하지 못하는 문제가 발생하고 있습니다.

특히 실제 사례에서 '국민취업지원제도'를 몰라 신청하지 못한 경험을 통해 문제의식을 갖게 되었고, **사용자 조건에 맞춘 공공서비스를 추천해주는 AI 챗봇**의 필요성을 느끼게 되었습니다.

## 2. 프로젝트 목표

- **맞춤형 서비스 추천**: 연령, 지역, 직업, 관심 분야 등 사용자 조건 기반 추천
- **대화형 챗봇 인터페이스**: 자연어 입력 기반 질의응답 시스템 구현
- **공공데이터 연동**: 정부24, 고용24, 온통청년 등의 API 및 문서 활용
- **실시간 응답 시스템**: WebSocket 기반 챗봇 구현

## 3. 주요 기능

**핵심 기능**

- 회원가입 및 로그인 (JWT 인증, 소셜 로그인 포함)
- 마이페이지, 프로필 정보 설정 (관심사, 지역 등)
- 사용자 조건 기반 맞춤형 정책 추천
- 챗봇을 통한 실시간 질의응답
- 전국 공공서비스 정보 제공 및 검색
- 최신 정책 정보 자동 업데이트 (공공 API 연동)

**부가 기능**

- 채팅방 생성/삭제/입장 기능
- 프리미엄 서비스: 상세 분석 리포트, 결제 기능
- 이메일 인증, 공지메일 발송 (SMTP + Celery)

## 4. 기술 스택

| 구분 | 기술 |
| ---- | ---- |
| **백엔드** | Django REST Framework, Redis, Django Channels |
| **프론트엔드** | HTML, CSS, JavaScript (빌드 없이 배포) |
| **AI/LLM** | OpenAI GPT-4o mini, LangChain, CrewAI |
| **벡터 DB** | ChromaDB |
| **관계형 DB** | PostgreSQL |
| **API 연동** | 정부24, 고용24, 온통청년 등 공공 API |
| **기타** | WebSocket 실시간 통신, Docker, GitHub Actions (CI/CD), PortOne |

## 5. AI 구조 및 설계 방식

AI 챗봇 기능은 다음 흐름으로 구성했습니다:

- **RAG 구조**: 사용자의 질문에 답하기 전, 벡터 DB에서 관련 정보를 검색하고 그걸 바탕으로 답변을 생성합니다.
- **LangChain**을 활용해 검색과 생성 체인을 구성했고,
- **ChromaDB**를 벡터 저장소로 사용해 유사 문서 검색이 가능하도록 했습니다.
- 일부 고급 기능은 **CrewAI 기반 멀티 에이전트 구조**를 시도해봤습니다.
